## 核心定义：一人公司的本质

**不是一个人在工作，是一个人建了一套系统，系统在工作。**

作者的状态：
- 没有团队，没有固定办公室，没有固定上班时间
- 收入来自自己建立的系统，不是雇主
- 内容自动生成，站点自动监控，外链自动检查
- 定期检查是否偏离方向，偶尔换零件，偶尔加 cron 任务

**护城河根本是信息不对称** → 不能告诉任何人你在做什么

## 第一件事：想清楚盈利模式

**流量是前提，不是结果。**

没有变现路径的流量只是一堆待转化的可能性。

很多人做站群的困境根源：**搞混了"流量业务"和"盈利业务"**

## 流量最好的四种站型

### 1. 测评站
- 用户在做购买决策时需要对比信息
- "best X"、"X vs Y"、"X review" 转化率最高
- 用户处于购买漏斗中后段
- 内容要求：真实横向对比、具体使用场景、诚实陈述优缺点、明确推荐结论

### 2. 攻略站
- 用户遇到具体问题时搜索解决方案
- 用户黏性极高，复访率高
- 长尾关键词、意图明确、竞争小
- 有天然更新需求（版本更新、新内容、政策变化）

### 3. 小游戏站
- 停留时间10-20分钟，跳出率极低
- 页面停留时长是算法极度偏爱的信号
- 开发门槛不高，很多开源项目可用

### 4. 工具站
- 单位换算、文字处理、图片压缩、颜色选择器等
- 搜索意图精准，收藏/书签行为高
- 复访率是内容站的几倍
- 可做订阅版、API付费调用

**共同点**：本身就是好内容，不需要靠技巧骗排名

## 盈利模式的层次

| 层次 | 类型 | 特点 | 作者态度 |
|------|------|------|----------|
| 底层 | 广告收入（CPM） | 简单粗暴，天花板极低，损耗用户体验 | ❌ 不做 |
| 中间层 | 轴文合作 | 收益高，做好了对读者有价值 | ⭐ 只做软文，不做硬广 |
| 高层 | SaaS/订阅收入 | 复利机制，月复一月稳定订阅 | ⭐⭐⭐ 被严重低估 |

**软文规则**：去掉推广意图，内容仍有价值 = 软文；去掉推广什么都不剩 = 硬广

## Ghost博客：被低估的订阅收入工具

**核心价值**：把内容本身变成产品，不是把内容当作获取流量的中间环节

优势：
- 自带会员和订阅功能
- 免费内容吸引流量，付费会员获独家内容
- 收入不受算法更新影响
- 官方托管每月$9起，支持自定义域名、完整SEO控制
- 自托管版本免费，只需服务器费用

## 站群的边界比你想的宽

**本质**：你控制的、能产生SEO价值或流量价值的网络节点总和

| 平台 | 价值 |
|------|------|
| Ghost博客 | 自定义域名 + SEO控制 + 订阅系统 |
| Medium | 极高域名权重 + 算法分发 + canonical链接回主站 |
| WordPress.com | 子域名有较高权重，免费托管 |
| Google Sites | 收录极快，几小时内出现在索引 |
| Blogger (blogspot.com) | 历史权重可观，收录优先级高 |

**正确逻辑**：重要内容在自建站，需要快速曝光用平台做辅助

**canonical链接**：告诉Google"权威版本在主站，这里是转载"，利用平台流量分发但权重不分散

## SEO从未死去，外链从未失效

**2026年结论**：
- SEO依然是内容站最重要的流量来源
- 外链依然是权重最高的单一因素

**机制决定**：
- PageRank逻辑没被颠覆：其他值得信任的网站愿意链接到你 = 内容有价值
- 2024年Google内部文件泄露确认：高质量外链权重远高于技术SEO和内容质量

**AI搜索强化了这个逻辑**：
- ChatGPT、Perplexity、AI Overview引用来源高度集中在传统SEO排名靠前的站点
- AI可见性跟着SEO好，SEO差 = AI搜索透明

**SEO三个层次优先级**：

| 层次 | 内容 | 优先级 |
|------|------|--------|
| 第一层 | 技术SEO（页面速度、CWV、移动端、结构化数据） | 底线，不做好其他是沙子 |
| 第二层 | 内容SEO（关键词、意图匹配、E-E-A-T） | 质量门槛 |
| 第三层 | 外链SEO（数量、质量、多样性、锚文字） | 决胜因素 |

**误区**：很多人精力全放在内容，忽视外链 → 在高竞争垂类打不过外链更强但内容未必更好的对手

**流量来源不只是Google**：
- Google 60%+
- Bing 15%-25%（Copilot整合后份额增长）
- DuckDuckGo、Yahoo（Bing后台）、Yandex特定地区

**建议**：自建统计系统，不依赖第三方工具（数据主权）

## 2026年站点网络的真实状态

**低质量站群已死** → SpamBrain识别能力显著提升，处罚更快

**高质量站群特征**：

1. **域名质量是基础**
   - Trust Flow 15以上
   - linking root domains 20以上
   - 无处罚记录
   - Wayback Machine真实内容历史
   - 过期域名价格涨了3倍

2. **内容质量不能偷懒**
   - 达到"如果是独立站也有存在价值"标准
   - 不是填充内容，不是批量生成
   - 文章风格、体裁、语气自然变化

3. **平台多样性是护城河**
   - 异构节点生态：自建站 + 高权重平台 + 工具站 + 小游戏站
   - 没有统一模式 = 没有可被整体识别的结构性弱点

4. **主机分散是底线**
   - 不同国家主机商
   - 不同C类IP段
   - 不同DNS服务商（不要用同一个Cloudflare账号）
   - 不同域名注册商

5. **绝对不卖外链**
   - 短期收益，长期必死
   - 开始卖的那一刻，网络不再是私人的

**最高标准**：让网络里每一个站都不需要依靠网络才能存活

## Token成本优化三法

Token是这个时代的原材料（就像制造业需要钢铁）

### 方法一：云厂商创业赠金
- 申请初创企业赠金计划
- 通过率60%-70%
- 额度从几百到几千美元

### 方法二：NewAPI统一路由层

价值一：根据任务类型自动分发给最合适模型
- 批量内容生成 → 便宜小模型
- 深度推理判断 → 大模型
- 整体成本压缩30%-50%

价值二：统一管理多账号、多平台API key

### 方法三：API中转生态
- 硅基流动、七牛云邀请注册 → 批量额度进NewAPI池
- GPT Team订阅反向代理：成本最低方案之一

**注意**：IP池质量比价格更重要

**核心逻辑**：Token成本是可变成本，系统设计是固定成本 → 固定成本多花时间，可变成本长期复利

## 系统骨架：Linux和定时任务

**最可靠的一层**：cron表达式，每天准时跑，不受节假日、心情、产品更新影响

**cron树示例**：
| 时间 | 任务 |
|------|------|
| 凌晨2点 | 关键词排名监控 |
| 凌晨3点 | 内容生产队列（发布时间/数量随机偏移） |
| 凌晨4点 | 外链健康检查 |
| 早8点 | 流量数据汇总（自建统计系统） |
| 每4小时 | 站点健康检查（HEAD请求） |
| 每晚 | Agent状态更新（提取观点档案） |

**内容发布细节**：
- 发布时间和数量不能固定 → 模拟真实博主不规律节奏
- 文章数量宁少勿多 → 一篇高质量远大于十篇平庸
- 不达标的草稿直接丢弃，不发布

**自建统计系统原因**：
1. 账号数量问题（站群规模大）
2. 数据主权问题（流量模式、用户行为不应交给Google）

## 工程哲学：够用就行，然后蒸馏

**原则**：先凑合，后蒸馏，全程模块化

**模块化标准**：换掉任何一块，系统不会整体崩

- API挂了 → 换接口
- key用完 → 换key
- 脚本改了 → 只改那个脚本
- 站调整 → 只改那个站配置

**痛点驱动蒸馏**：先跑两个月，积累"这个地方很烦"的感受，再精准处理

**残忍现实**：精心设计的架构面对会动的靶 → API改格式、平台调权限、真实环境跑三天发现完全不是预想

**美学丑但跑得稳** > 完美设计但没跑起来

## Agent的角色扮演和成长曲线

每个站配备AI Agent作为运营者（角色，不只是工具）

**设定内容**：
- "作者"是谁，有什么背景
- 在领域有什么独特视角和偏好
- 写作风格

**成长曲线**：
- 刚建站：入门向内容，保守选题，分享者语气
- 内容积累：细分话题，引用自己之前内容，建立观点体系
- 历史支撑的评价、前后文验证的推荐

**一致性**：立场和观点一致，不是写作风格固定

**写作形式必须变化**：深度长文、快速对比、问答、数据报告...体裁随内容需求变化

**如果所有文章像同模板出来** → 被识别为机器生成特征

**E-E-A-T**：靠真实使用经历描述、具体数据案例、明确立场理由

## GEO：写给AI读的优化层

Generative Engine Optimization：让内容被ChatGPT、Perplexity、Gemini引用

**AI引擎读取方式**：段落级检索 → 每个H2/H3下第一句必须是核心观点直接陈述

**数据威力**：包含可溯源统计数据的内容，AI引用率高30%-40%

**四项实践**：

1. **开放AI爬虫**
   - Cloudflare 2024默认屏蔽AI爬虫
   - 明确放行 GPTBot、ClaudeBot、PerplexityBot

2. **每页Markdown版本**
   - URL后加.md访问纯文本
   - AI爬虫更倾向读干净格式

3. **结构化FAQ模块**
   - 底部FAQ区块 + schema markup
   - AI偏向引用问答格式内容

4. **llms.txt文件**
   - 根目录放llms.txt告诉AI爬虫主题和核心内容位置
   - 辅助信号，不是万能开关

**叠加优势**：小语种站GEO竞争密度比英文站更低

## 必须有一个真正赚钱的核心业务

**站群是放大器，不是引擎**

核心业务定义：独立于站群、由自己控制、不完全依赖搜索算法的生意
- 独立站（有真实复购）
- 服务性质收入
- 付费社群
- SaaS工具

**为什么重要**：
- 算法会更新，不提前通知
- 流量可一次更新下降50%
- 所有收益绑在站群上 = 存在性风险
- 有独立核心业务 → 抗风险基础

---

## 铁律：永远不要透露站群域名

**一个都不行**。不是员工、朋友、合作伙伴、同行、师傅、家人。

**四个原因**：

1. 合作伙伴可能某天开始做自己的站，脑子里有你的站地图
2. 同行可能在私下讨论提到你域名，讨论里有其他人
3. 某个站被公开关联 → 对外独立性破了 → 变成已知网络节点
4. 最难防的不是恶意，是无意（截图露出一角、聊天顺嘴提到）

**操作卫生**：把"不说"变成默认值，不是需要提醒自己的规则
