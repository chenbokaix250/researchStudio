## 核心问题

**第二大脑为什么总是烂尾？**

传统第二大脑的死亡循环：
- 搭建时认真：标签打好、结构理清、交叉链接做好
- 两个月后：标签过时、断链没人修、新笔记乱扔
- 再两个月：一堆垃圾 → 重建 → 再烂尾

**根本原因：维护成本太高，比写笔记本身还累**

---

## Karpathy 方法论

Andrej Karpathy（OpenAI创始团队、前Tesla AI总监）提出的方案：

> "用 LLM 维护个人知识库，让 token throughput 从操作代码转向操作知识"

**核心思路：让 Claude Code 替你维护 Wiki**

### 三层架构

1. **raw/** - 原始素材（文章、笔记）
2. **wiki/** - AI整理后的知识页面
3. **规则文件** - 告诉AI怎么维护

### 三种操作

1. **录入** - 扔素材进去，AI自动整理
2. **查询** - AI读Wiki回答问题
3. **修复** - AI自动检查并修复问题

## 实战步骤

### Step 1: 搭建Vault（5分钟）

1. Obsidian 建一个 Vault
2. Claude Code 打开这个 Vault 文件夹
3. 贴入 Karpathy 规则文档：
   - 地址：https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f

Claude Code 自动创建：
- `raw/` - 原始素材目录
- `wiki/` - 知识页面目录
- `index.md` - 索引文件
- `log.md` - 操作历史

### Step 2: 录入第一篇文章

**录入方式：**
- 手动：Obsidian Web Clipper 插件裁剪
- 自动：Hermes等Agent持续收集

**Claude Code 命令：**
```
帮我录入最新文章
```

**AI 自动完成：**
- 通读文章 → 写摘要
- 拆解概念 → 每个概念建页面
- 提取人物/工具/方法 → 建页面
- 双向链接 → 自动关联
- 更新 index.md → 登记
- 记录 log.md → 操作历史

**结果：一篇文章 → wiki/ 多出十几个页面**

### Step 3: 维护（零人工）

**健康检查命令：**
```
帮我检查一下 Wiki 的健康状况
```

**AI 检查四种问题：**
1. 页面矛盾 - 说法不一致
2. 孤儿页面 - 没有链接指向
3. 缺失概念 - 反复提到但无页面
4. 过时信息 - 被新素材推翻

**自动化：设为定时任务，每周自动跑一次**

## 核心价值

### 1. 结构化知识图谱

打开 Obsidian 图谱视图：
- 点 = 知识页面
- 线 = 引用关系
- 知识自动形成网络

### 2. AI 深度理解你

**让 AI 使用你的 Wiki：**

- **临时方式**：把 wiki/index.md 丢进 AI 聊天窗口
- **永久方式**：写入 Agent 配置文件（如 CLAUDE.md）

```
需要了解我的时候去读 wiki/index.md
```

**效果：**
- AI 知道你的风格、偏好、踩过的坑
- 用得越久，越懂你
- 不用每次从零解释

### 3. 学习新领域

扔一个话题进去：
- AI 拆解成子概念
- 每个子概念建页面
- 顺着页面学习

### 4. 放入个人思考

最独有的东西：
- 日常思考
- 复盘总结
- 判断推理

互联网搜不到，AI拿到后会越来越懂你。

### 5. Token 高效

分级查询：
- AI先读index找相关页面
- 再读具体内容
- 不用把所有笔记全塞进去

## 核心洞见

### 为什么这次不会烂尾？

传统：维护成本 = 你的时间（标签过时、断链修复、结构重组）

现在：维护成本 = 一条命令（AI自动体检、修复）

### 复利效应

知识库是复利的：
- 每多录一篇 → 后面所有查询/输出都变好一点
- 搭的人 vs 不搭的人 → 三个月后差距拉开
