## 核心观点

Skill 就是给 AI 准备的**技能手册**，核心只需要一个 Markdown 文件（SKILL.md），不需要写代码。

---

## 一、Skills 到底是什么

- **比喻**: Skill = AI 的菜谱。告诉 AI 碰到某类任务该怎么做、用什么流程、输出什么格式。
- **结构**:
  ```
  my-skill/
  ├── SKILL.md      # 核心：告诉 AI 怎么干活
  ├── scripts/      # 可选：需要跑的脚本
  └── references/   # 可选：参考文档、模板
  ```
- **SKILL.md 两部分**:
  1. **元数据**（metadata）：名字、描述、触发条件
  2. **操作指南**：告诉 AI 具体怎么一步步干活
- **关键细节**: Description 决定 AI 能否正确匹配到你的 Skill，写得准才能被触发。

---

## 二、5步从0造一个Skill

| 步骤 | 内容 |
|------|------|
| ① 找到重复劳动 | 判断标准：这件事重复做了 3 次以上，每次流程差不多 |
| ② 先手动跑通 | 自己先手动做一遍完整流程，把每一步写下来，这是最好的需求文档 |
| ③ 把需求讲给 AI | 用大白话描述需求，AI 会追问细节，然后生成 SKILL.md |
| ④ 跑起来，改起来 | 试跑，改 2-3 轮基本稳定。前两轮修大问题，第三轮调细节 |
| ⑤ 分享出去（可选） | 发布到 GitHub 或 ClawHub |

## 三、实战案例：AI 日报 Skill

### 起因
每天从 20+ 信息源扒 AI 新闻（VentureBeat、TechCrunch、OpenAI Blog、HuggingFace Papers、GitHub Trending 等），耗时 40+ 分钟。

### 最终效果
一句"AI 日报"自动跑：
- 从 20+ 源并发抓取
- 多 subagent 并行处理分类
- 汇总生成 Markdown
- 渲染成 Newsletter 风格 HTML
- 自动截图并按分类切割

### 踩坑记录
1. **Hacker News 内容太杂** → 直接砍掉这个源
2. **长图微信看不清** → 按分类切割成多张
3. **GitHub Trending 误判** → 加 README 内容验证是否真和 AI 相关

## 四、实战案例：站在别人肩膀上改配图 Skill

### 发现宝玉的方案
宝玉开源的 `baoyu-article-illustrator`：
- 提示词规范做得好（每种配图类型有结构化模板）
- 配图类型匹配逻辑清晰（数据→infographic，流程→flowchart，对比→comparison）
- 工作流串得很顺

### 改动点
宝玉版本锁定了预设视觉风格 → 作者改为**把风格决策权交给 AI**，让 AI 根据文章类型自主判断适合什么风格。

### 启示
造 Skill 最快的路径：找靠谱的基础版本，改成私人定制版本。

## 五、踩坑建议

| 建议 | 说明 |
|------|------|
| Description 要写准 | 一句话说清楚干什么，太宽泛/太窄都不行 |
| 先手动跑通再封装 | 手动都没做过，直接让 AI 生成大概率不能用 |
| 一个 Skill 解决一个问题 | 别贪多，大任务拆成多个 Skill |
| 迭代心态 | 第一版一定不完美，改 2-3 轮就稳了 |

---

## 六、三个推荐安装的 Skills

| Skill | 功能 |
|-------|------|
| **find-skills** | 技能发现器：自动搜索、推荐、帮你安装 ClawHub 上的 Skill |
| **skill-creator** | 技能创建器：用自然语言描述需求，自动生成 SKILL.md |
| **skill-vetter** | 技能安全审查器：装第三方 Skill 前检测红旗信号（可疑权限、异常命令、不明外部调用） |

**组合逻辑**: 有现成的用 find-skills 找 → 找不到用 skill-creator 造 → 装之前让 skill-vetter 扫一遍。

---

## 总结

从每天 40 分钟扒新闻，到一句话搞定日报，中间隔的就是一个 SKILL.md 文件。

如果你也有重复性工作让你头疼：
1. 先从 find-skills 搜有没有现成的
2. 找不到就自己造，跟着 5 步走
3. 一个 Skill 解决一个具体问题，慢慢攒着，AI 就越来越顺手
