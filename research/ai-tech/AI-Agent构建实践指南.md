## 核心框架

### Agent = Role + Goal + Tools + Rules + Output format

### 工作循环
```
用户输入 → LLM思考 → 决策(响应/调工具) → 执行反馈 → 循环
```

### 三要素：LLM(大脑) + Tools(双手) + Memory(笔记本)

---

## 五种 Workflow 模式

| 模式 | 原理 | 适用场景 |
|------|------|---------|
| Prompt Chaining | 顺序步骤，每步处理前步输出 | 可分解任务 |
| Routing | 分类→路由到专门处理器 | 不同输入需不同处理 |
| Parallelisation | 多个LLM并行运行 | 子任务独立或需共识 |
| Orchestrator-Workers | 中央LLM动态分解委派 | 复杂不可预测任务 |
| Evaluator-Optimiser | 生成→评估→循环改进 | 有明确评估标准 |

## 实战原则

### 工具
- 更好的工具 ≠ 更多的工具
- 需外部数据/动作 → 用工具；否则不加
- 一个工具 = 一个清晰任务

### 记忆
- 70%场景不需要记忆
- 短期记忆：消息历史（默认已有）
- 长期记忆：文档/PDF（RAG）
- 能工作不加 → 不加

### 构建
- 从一个任务、一个Agent、一个提示词开始
- 最多1-2个工具
- 用真实混乱输入测试
- 逐项改进，不要一开始就复杂化

## 四问定 Agent

1. **结果是什么？** → 应产出什么
2. **需要什么信息？** → 搜索/文件/数据库？
3. **允许什么动作？** → 只回答/搜索/编辑/发邮件？
4. **遵守什么规则？** → 语气/格式/约束/安全

## 用 AI 设计 Agent 模板

```
目标：[描述]
用户会问：[5个真实示例]
能访问：[工具列表]
必须：[规则]
绝不：[边界]

生成：规格 + 系统提示词 + 工具列表 + 路线图 + 10测试案例
```
